读《程序员必读的职业规划书》有感

本文是《程序员必读的职业规划书》的读后感，感谢@Easy的赠书。当时收到快递GG的电话时还一头雾水，最近没有网购什么东西啊。这是我自上微博以来，通过转微博获得头一份赠品，还是如此有干货的一本好书，真是非常幸运。

以下是正文：

准备去找一份好工作了么？

如果你的回答是肯定的，那我们先从门卫的三大终极问题中的两个开始。

你是谁

你有清楚的自我认知么？你的人生目标是什么，职业规划又是啥？

对于我来说，大概就是写出一些有趣好玩以及有用的软件，同时也要赚尽可能多的钱。人类这种生物太复杂搞不明白，估计我也就能往专家线这个方向走了。能走多远不知道，但是我希望可以把代码一直写下去。

所以我的路线大概大概是这个样子的，如果顺利的话：

小工 -> 独立开发者 -> 成功的独立开发者 -> 技术合伙人

当然，如果前路曲折的话，可能是这个样子的：

小工 -> 独立开发者 -> 失业 -> 小工 -> 独立开发者 -> 失业 -> 小工 -> … -> 成功的独立开发者 -> 技术合伙人

如果你不清楚，可以看看这本书，里面提到了程序员职业的四大象限，也许能帮到你。你可以在原理篇与准备篇提供的帮助下，初步规划出你的职业道路。

你到哪里去

上一个问题里提到了大致的规划，这个问题里，需要把这个目标具体化了。

你的下一步是什么？怎么达成目标？

就我个人而言，接下来准备搭一下iOS开发的末班车，这跟我之前的方向截然不同。由于在上一家公司时并没有太多的时间学习相关的知识，我选择了脱产学习，唔，边做边学。希望能学习新知识的同时，把产品做出来。这样即使在产品不卖座的情况下，还能在简历上添上一笔相关从业经历。不至于像现在这样，每封简历都石沉大海。

如果你知道了下一步是什么，但是不知道怎么下手，这本书毫无疑问会成为你的助力。从怎么准备简历，到如何求职以及应付面试，操作篇里可都是满满的实践干货，不容错过。

最后

这本书不厚，如果不算后面那长长的全栈技能树，连100页都没有。但是如此薄的一本书，值得我们一再研读。其实不仅针对于程序员，对于其它职业也都有参考意义。如果你没看过这本书，请一定不要错过。

杂谈读书
